NNG

NNG es una biblioteca de sockets que proporciona varios patrones de comunicación comunes. Su objetivo es hacer que la capa de red sea rápida, escalable y fácil de usar. Implementado en C, funciona en una amplia gama de sistemas operativos sin más dependencias.

Los patrones de comunicación, también llamados "protocolos de escalabilidad", son bloques básicos para construir sistemas distribuidos. Al combinarlos, puede crear una amplia gama de aplicaciones distribuidas. Los siguientes protocolos de escalabilidad están disponibles actualmente:

PAIR: comunicación simple uno a uno
BUS: comunicación sencilla de varios a varios
REQREP: permite crear grupos de servicios sin estado para procesar las solicitudes de los usuarios.
PUBSUB: distribuye mensajes a grandes conjuntos de suscriptores interesados
PIPELINE: agrega mensajes de múltiples fuentes y los equilibra entre muchos destinos
ENCUESTA: permite consultar el estado de varias aplicaciones de una sola vez

SECCION 3 COMPAT: Funciones de Bibliotecas Compatibles

Esta sección documenta las funciones compatibles con la biblioteca nanomsg 1.0. Estas funciones se proporcionan como una ayuda de transición, para los desarrolladores de aplicaciones que llegan a NNG desde libnanomsg , y no se recomienda su uso en nuevas aplicaciones. Si bien esto se desaconseja para un uso a largo plazo, como ayuda de transición, las aplicaciones pueden usar el valor devuelto por nng_socket_id()en estas funciones como un descriptor de socket (como si el socket se abriera a través de nn_socket()). Este tipo de mezcla de API solo debe utilizarse durante la transición de la API heredada a la nueva API.

SECCION 3 http: Funciones HTTP Complementarias

Esta sección documenta las funciones suplementarias de soporte HTTP (Protocolo de transporte de hipertexto) que están disponibles. Estas funciones se pueden utilizar junto con el transporte WebSocket para protocolos de escalabilidad, o se pueden utilizar para construir otros tipos de aplicaciones que se comunican mediante HTTP. También es posible combinar los dos, de modo que un servidor HTTP que proporcione contenido estático o dinámico también se pueda utilizar para alojar uno o más sockets de protocolos de escalabilidad. Actualmente, NNG solo admite HTTP / 1.0 y HTTP / 1.1.

SECCION 3tls: Funciones TLS Suplementarias

Esta sección documenta las funciones suplementarias de TLS (seguridad de la capa de transporte) que están disponibles. El soporte TLS está disponible cuando se utilizan protocolos de escalabilidad con el transporte TLS o cuando se usa WebSocket, ya sea con el transporte WebSocket para protocolos de escalabilidad o combinado con otras capacidades HTTP. Sin embargo, estas funciones dependen del soporte de la biblioteca que no se incluye directamente con NNG , por lo que su presencia dependerá de si este soporte adicional estaba presente y habilitado con libnng . Actualmente, este soporte adicional puede ser proporcionado por la biblioteca mbedTLS o por complementos externos. Comuníquese con Staysail Systems, Inc. para obtener detalles sobre las opciones disponibles comercialmente, incluido el soporte para criptografía validada FIPS 140-2 y TLS v1.3.

SECCION 7: Protocolos y Transportes

Esta sección documenta varios protocolos y transportes que están disponibles en la distribución. Los protocolos implementan patrones de comunicación, como solicitud / respuesta, publicación / suscripción, etc. Un socket dado se crea con exactamente un protocolo, y ese protocolo define el comportamiento clave del socket. Por el contrario, los transportes son los mecanismos subyacentes mediante los cuales los mensajes se mueven entre los participantes, como TCP / IP o IPC de dominio UNIX. Un enchufe determinado puede utilizar varios transportes al mismo tiempo.
