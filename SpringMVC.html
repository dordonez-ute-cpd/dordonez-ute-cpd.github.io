Modelo-vista-controlador

Model-view-controller (normalmente conocido como MVC) es un patrón de diseño de software comúnmente utilizado para desarrollar la interfaz de usuario que divide la lógica de programa relacionada en tres elementos interconectados. Esto se hace para separar las representaciones internas de la información de las formas en que la información se presenta y acepta del usuario. Este patrón se ha vuelto popular para diseñar aplicaciones web,que se utilizan tradicionalmente para interfaces gráficas de usuario de escritorio(GUI). Los lenguajes de programación populares como JavaScript, Python, Object Pascal/Delphi, Ruby, PHP, Java, C,y Swift tienen frameworks MVC que se utilizan para el desarrollo de aplicaciones web o móviles directamente de la caja.

Modelo

El componente central del patrón. Es la estructura de datos dinámica de la aplicación, independientemente de la interfaz de usuario. Gestiona directamente los datos, la lógica y las reglas de la aplicación.

Vista

Cualquier representación de información como un gráfico, diagrama o tabla. Varias vistas de la misma información son posibles, como un gráfico de barras para la administración y una vista tabular para los contadores.

Controlador

Acepta la entrada y la convierte en comandos para el modelo o la vista.Además de dividir la aplicación en estos componentes, el diseño modelo-vista-controlador define las interacciones entre ellos

  • El modelo es responsable de administrar los datos de la aplicación. Recibe la entrada del usuario del controlador.
  • La vista significa la presentación del modelo en un formato determinado.
  • El controlador responde a la entrada del usuario y realiza interacciones en los objetos del modelo de datos. El controlador recibe la entrada, opcionalmente la valida y, a continuación, pasa la entrada al modelo.

Spring MVC Creación de aplicaciones web

Spring Web MVC es un sub-proyecto Spring que esta dirigido a facilitar y optimizar el proceso creación de aplicaciones web utilizando el patrón MVC (Modelo-Vista-Controlador), donde el Modelo representa los datos o información que manejará la aplicación web, la Vista son todos los elementos de la UI (Interfaz de Usuario), con ellos el usuario interactúa con la aplicación, ejemplo: botones, campos de texto, etc., finalmente el Controlador será el encargado manipular los datos en base a la interacción del usuario.

Spring Boot DevTools

Una característica común del desarrollo de aplicaciones web es codificar un cambio, reiniciar la aplicación y actualizar el explorador para ver el cambio. Todo este proceso puede comer mucho tiempo. Para acelerar este ciclo de actualización, Spring Boot ofrece un práctico módulo conocido como spring-boot-devtools. Spring Boot Devtools:

  • Permite intercambio en caliente.
  • Cambia los motores de plantilla para deshabilitar el almacenamiento en caché.
  • Permite que LiveReload actualice automáticamente el explorador.
  • Otros valores predeterminados razonables basados en el desarrollo en lugar de la producción.
